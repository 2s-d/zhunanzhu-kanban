# 专注项目 - 数据看板需求文档

## 项目概述

**专注 (Focus)** 是一个 Flutter 开发的 Android 专注学习应用，通过科学的休息机制和奖励系统帮助用户提高学习效率。

## 核心数据模型

### 1. Project (学习项目)
- `id`: 项目唯一标识
- `name`: 项目名称
- `createdAt`: 创建时间
- `totalStudyMinutes`: 累计学习时长（分钟）
- `totalPointsTenths`: 累计运（以0.1为单位，例如100.0运 = 1000 tenths）
- `logs`: 项目日志列表（最多500条）
- `rewardShortRest`: 短休息奖励（1-10，可选）
- `rewardLongRest`: 长休息奖励（20-100，可选）
- `rewardPerMinuteTenths`: 每分钟奖励（1-30 tenths = 0.1-3.0运，可选）

### 2. AppData (应用全局数据)
- `schemaVersion`: 数据版本（当前为1）
- `globalPointsTenths`: 全局总运（所有项目+签到累计）
- `projects`: 项目列表
- `lastCheckInDate`: 上次签到日期
- `lastFortuneDate`: 上次抽运势日期
- `todayFortune`: 今日运势（DailyFortune对象）
- `themeSeedColorValue`: 主题色值

### 3. LogEntry (日志条目)
- `message`: 日志消息
- `timestamp`: 时间戳
- `type`: 日志类型（info/success/warning/rest）

### 4. DailyFortune (今日运势)
- `level`: 运势等级（ping/xiaoJi/daJi）
- `date`: 日期
- `message`: 运势消息

## 关键业务规则

### 学习模式

**自由模式：**
- 从0开始计时
- 学习时长实时计入项目累计
- 达到7200分钟（5天）自动结束
- 每分钟获得0.1运（包括休息时间）

**预设模式：**
- 设置60-720分钟的学习计划
- 学习时长只在任务完成时计入
- 完成任务额外+30运
- 未完成任务不计入时长

### 休息机制

**小阶段休息：**
- 每3-5分钟随机触发一次
- 休息10秒，自动继续
- 每次+1运（可配置1-10）
- 休息期间暂停有效学习时间

**大阶段休息：**
- 每90分钟触发一次
- 休息20分钟，自动继续
- 每次+30运（可配置20-100）
- 休息期间暂停有效学习时间

### 奖励规则

1. **每分钟奖励**：0.1运/分钟（可配置0.1-3.0）
   - 基于总会话时间（包括休息）
   - 实时累计

2. **短休息奖励**：+1运（可配置1-10）
   - 基于有效学习时间触发

3. **长休息奖励**：+30运（可配置20-100）
   - 基于有效学习时间触发

4. **预设完成奖励**：+30运（固定）
   - 仅在预设模式完成任务时

5. **签到奖励**：+30运/天
   - 支持补签（漏签天数×30）

### 数据持久化

- 数据保存在 `app_data.json` 文件
- 位置：应用文档目录
- 格式：JSON，带版本号（schemaVersion: 1）
- 包含：全局运、项目列表、签到日期、运势、主题色

## 关键指标

### 项目级指标
- `totalStudyMinutes`: 项目累计学习时长
- `totalPointsTenths`: 项目累计运
- `logs.length`: 日志数量（最多500）

### 全局指标
- `globalPointsTenths`: 全局总运
- `projects.length`: 项目总数
- `lastCheckInDate`: 最后签到日期
- 连续签到天数（需计算）

### 时间指标
- 今日学习时长
- 本周学习时长
- 本月学习时长
- 平均每日学习时长

## 数据流转

```
用户操作 → 业务逻辑 → 数据更新 → 持久化保存
    ↓           ↓          ↓           ↓
开始学习 → 计时器启动 → 实时累计运 → 保存到JSON
休息触发 → 奖励计算 → 增加运 → 保存到JSON
完成任务 → 结算逻辑 → 更新时长 → 保存到JSON
```

## 数据看板建议展示内容

### 1. 全局概览
- 总运数（globalPointsTenths / 10）
- 项目总数
- 总学习时长（所有项目累计）
- 连续签到天数

### 2. 项目分析
- 每个项目的学习时长
- 每个项目的运获得
- 项目创建时间
- 项目活跃度（最近学习时间）

### 3. 时间趋势
- 每日学习时长趋势图
- 每周学习时长统计
- 每月学习时长统计
- 学习时段分布（如：上午/下午/晚上）

### 4. 奖励分析
- 运获得来源分布（签到/学习/休息）
- 休息次数统计
- 每分钟奖励累计
- 奖励配置使用情况

### 5. 学习习惯
- 平均单次学习时长
- 最长连续学习时长
- 休息频率分析
- 学习模式偏好（自由/预设）
